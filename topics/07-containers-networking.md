# 07. Контейнеры, виртуализация, сети

## Контейнеры и виртуализация

**В: Контейнеризация vs виртуализация?**
О: VM эмулирует железо и поднимает свою гостевую ОС (тяжёлая, сильная изоляция, медленнее старт). Контейнер использует **ядро хоста**, изолирует процессы/сеть/ФС (лёгкий, быстрый старт, изоляция слабее).

**В: К чему привязан контейнер?**
О: К ядру хостовой ОС. Классические контейнеры — Linux (namespaces, cgroups, capabilities). На Windows/macOS — через прослойку Linux VM (WSL2/виртуализация).

**В: Что ставится вместе с Docker?**
О: Docker Engine (движок управления образами/контейнерами/сетями), Docker Desktop (оболочка), и лёгкая Linux-среда (WSL2/VM) для Linux-контейнеров.

**В: Как смотреть поды в Kubernetes?**
О: Через API Kubernetes (kube-apiserver), обычно `kubectl get pods` / `kubectl describe pod`, доступ через kubeconfig. К поду обращаются не напрямую, а через API-сервер.

## Сети

**В: Что такое модель OSI?**
О: 7 уровней (снизу вверх): Физический → Канальный → Сетевой → Транспортный → Сеансовый → Представления → Прикладной. Помогает локализовать проблему по уровням.

**В: Канальный vs сетевой уровень?**
О: ⚠ Канальный — MAC-адреса, кадры, доставка внутри локальной сети. Сетевой — IP-адреса, **маршрутизация между сетями** (маршрутизация — это сетевой уровень, не канальный).

**В: Транспортный уровень?**
О: TCP (надёжная доставка с подтверждением, упорядочивание) и UDP (быстрее, без гарантии доставки), порты.

**В: Прикладные протоколы?**
О: HTTP/HTTPS, WebSocket, FTP/SFTP, SMTP/IMAP/POP3, DNS, SSH, NTP.

**В: Что такое DNS?**
О: Сопоставляет доменное имя ↔ IP (имя → IP). «Телефонная книга интернета».

**В: Что такое DHCP?**
О: Автоматически раздаёт сетевые настройки: IP, маску, шлюз, DNS. Убирает ручную настройку и конфликты IP.

**В: Что такое NAT?**
О: Подмена приватных IP на публичный при выходе в интернет. Позволяет нескольким устройствам выходить через один внешний IP.

**В: Что такое ARP?**
О: Сопоставляет IP ↔ MAC внутри локальной сети (IP → MAC). При неизвестном MAC — broadcast-запрос. Результат кэшируется.

**В: Шпаргалка «кто что делает»:**
О:
- **DNS**: имя → IP
- **ARP**: IP → MAC (в локалке)
- **DHCP**: выдать настройки сети устройству
- **NAT**: внутренний IP → внешний IP

**В: HTTP/2 vs HTTP/1.1?**
О: HTTP/2 — мультиплексирование (несколько запросов по одному соединению), сжатие заголовков, приоритизация. Поверх TCP, обычно с TLS (HTTPS).

**В: gRPC vs HTTP/REST?**
О: gRPC — RPC поверх HTTP/2 с protobuf (бинарный, компактный, быстрый, стриминг из коробки), для межсервисного общения. REST/HTTP — текстовый JSON, проще дебажить, для внешних API.
