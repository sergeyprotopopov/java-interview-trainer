# 09. Java Core: исключения, рефлексия, коллекции

## Общее

**В: Что такое Java?**
О: Объектно-ориентированный язык и платформа, работающая через JVM. Кроссплатформенность, строгая типизация, автоматическая сборка мусора, JIT-компиляция, богатая экосистема (Spring, Hibernate). Для backend, highload, enterprise.

## Исключения

**В: Какие два корневых типа исключений?**
О: `Exception` (прикладные ошибки) и `Error` (системные/JVM-проблемы, обычно не ловят).

**В: Checked vs unchecked?**
О:
- **Checked** — компилятор требует обработать/пробросить (`throws`): `IOException`, `SQLException`. Для ожидаемых проблем с внешними ресурсами.
- **Unchecked** (`RuntimeException`) — компилятор не требует: `NullPointerException`, `IllegalArgumentException`, `IndexOutOfBoundsException`. Для ошибок логики/входных данных.
- **Error**: `OutOfMemoryError`, `StackOverflowError`.

## Рефлексия и аннотации

**В: Что такое рефлексия?**
О: Доступ к метаданным и структуре (классы, методы, поля) в рантайме и динамический вызов. Нужна фреймворкам (Spring, Hibernate, Jackson), DI, сериализации. Минусы: медленнее, ломает инкапсуляцию, сложнее отлаживать.

**В: Аннотации в рантайме?**
О: `RetentionPolicy`: `SOURCE` (только компиляция), `CLASS` (в `.class`, не в рантайме), `RUNTIME` (доступна через reflection).

## Ключевые слова и модификаторы

**В: final / finally / finalize?**
О: `final` — запрет изменения (переменная/метод/класс). `finally` — блок, выполняется почти всегда после try/catch (очистка ресурсов; может не выполниться при `System.exit()`). `finalize` — устаревший метод очистки перед GC.

**В: Что значит final class?**
О: Класс нельзя наследовать (например, `String`). Для неизменяемости и предсказуемости.

**В: Модификаторы доступа?**
О: `private` (только класс), `default`/package-private (только пакет), `protected` (пакет + наследники), `public` (везде).

## Коллекции

**В: Что происходит при добавлении в ArrayList?**
О: Если есть capacity — запись в конец (O(1)). Если нет — resize: новый массив (~×1.5), копирование старых (O(n)). Амортизированно O(1). Обычный массив не расширяется.

**В: Как «сжать» массив/список?**
О: Массив — создать новый меньшего размера и скопировать. `ArrayList` — `trimToSize()`.

**В: Что такое Arrays.asList?**
О: Обёртка над массивом фиксированного размера. `set` работает (меняет исходный массив), `add`/`remove` → `UnsupportedOperationException`.
