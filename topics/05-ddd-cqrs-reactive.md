# 05. DDD, CQRS, реактивность

## DDD (Domain-Driven Design)

**В: Что такое DDD?**
О: Проектирование от предметной области, а не от таблиц. Общий язык (ubiquitous language) бизнеса и разработки, деление на bounded contexts. Полезно в сложных доменах и больших командах.

**В: Что такое bounded context?**
О: Ограниченный контекст, где термин имеет одно конкретное значение. В разных контекстах один термин («клиент») может означать разное. Снижает связность.

**В: Что моделируют внутри домена?**
О:
- **Сущность (Entity)** — объект с идентичностью (`User`, `Order`).
- **Value Object** — без идентичности, важно значение (`Money`, `Address`, `Period`).
- **Агрегат** — группа объектов с корнем (aggregate root), через который идут все изменения.
- **Доменный сервис** — логика, не ложащаяся на одну сущность.
- **Доменные события** — `OrderCreated`, `PaymentCompleted`.

**В: Принцип связности?**
О: Высокая связность внутри одной области, слабая — между областями. Взаимодействие через явные контракты, а не через внутренности соседней части.

## CQRS

**В: Что такое CQRS?**
О: Command Query Responsibility Segregation — разделение модели на запись (Command) и чтение (Query). Позволяет отдельно оптимизировать и масштабировать чтение/запись. Необязательно две БД — может быть просто разные модели/обработчики.

**В: Command или Query меняет состояние?**
О: ⚠ **Command** меняет состояние (`CreateUser`, `UpdateUser`, `DeleteUser`). **Query** только читает, без побочных эффектов (`GetUserById`).

**В: Когда CQRS оправдан?**
О: Большие системы с перекосом чтение/запись, сложные доменные модели, отдельные витрины для отчётов/поиска. На маленьких системах — избыточно.

## Реактивность

**В: Что такое WebFlux / реактивность?**
О: Реактивный неблокирующий стек Spring на `Mono`/`Flux`. Поток не простаивает на I/O. Полезен при большом числе конкурентных соединений и сетевого I/O. ⚠ Важно: вся цепочка должна быть неблокирующей, иначе выигрыша нет.

**В: WebFlux = микросервисы?**
О: Нет. Микросервисы — про архитектуру системы; WebFlux — про способ обработки запросов внутри сервиса.

**В: RxJava vs Flux?**
О: RxJava — общий реактивный инструмент (`Observable`, `Flowable`, `Single`, `Maybe`, `Completable`). Flux/Mono — реактивность в экосистеме Spring.
